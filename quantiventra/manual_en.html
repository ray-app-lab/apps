Section 01

Introduction

1-1. About this app

Quantiventra (a portmanteau of "Quantity" and "Inventory") is an Android app that streamlines physical inventory counting through barcode scanning using your smartphone's camera.

Simply point the camera at a barcode to automatically identify the item, enter the quantity, and save — transforming paper-based inventory into digital records. Results are stored as CSV files on the device for easy import into Excel or Google Sheets.

📦 Inventory Count

Quickly identify items via barcode scan and record quantities instantly. Built-in calculator makes summing across multiple locations easy.

🗂️ Master Management

Bulk-import master data via CSV. Pre-configure item names, part numbers, and theoretical stock levels.

📍 Location Management

Scan QR codes to set shelf locations. The shelf number is recorded in the CSV alongside each count.

📊 CSV Export

Scan results are automatically saved to device storage. Transfer to PC for aggregation and analysis.

Section 02

Installation & Initial Setup

2-1. Installation

Install the app from Google Play Store. After installation, tap the app icon to launch it.

2-2. First launch

  1. A camera permission dialog appears — tap Allow.

  2. After allowing camera access, you can start scanning immediately.

  3. If you want to use a master CSV, open Settings and tap Import Master CSV.

ℹ️
After setup Once camera permission is granted, you can start scanning, saving, importing a master CSV, and exporting data from the Settings screen.

2-3. How data is handled

The app manages master, history, and settings data automatically. In normal use, you do not need to know or handle the internal storage structure.

ℹ️
What the user needs to do Use Import Master CSV when you want to load master data, and use Export All Data or Share History when you want to take data out of the app.
Section 03

Creating & Importing Master CSV [Important]

✅
Preparing a master file greatly improves efficiency By importing a master CSV, item names and theoretical stock levels are automatically displayed when scanning. Without it, manual input is required each time.

3-1. Two Ways to Prepare the Master CSV (with Column Names)

There are two main methods to prepare your master file.

Method A: Create from scratch using Excel or Google Sheets

  1. Open Microsoft Excel or Google Sheets on your PC.

  2. Do not type the first-row headers manually. Copy and paste the header line exactly as shown in 3-3 Column Specifications (Column Names).

  3. Enter your item data starting from the second row.

  4. Save or download the file as "CSV (Comma delimited)" with the exact name quantiventra_master.csv (UTF-8 encoding is recommended).

Method B: Create a template in the app first (Beginner-friendly)

If starting from a blank CSV feels difficult, first register one sample item in the app, then export that CSV and use it as your editing template.

  1. Launch the app and register one sample item.

  2. Open Settings and run Export All Data so the master CSV is written out (see 9-2 Recommended backup procedure).

  3. Move the exported quantiventra_master_export.csv to your PC by mail, cloud storage, or direct copy.

  4. Open that CSV in Excel on your PC. Use the sample row as a guide, and paste data from any existing item table you already have into the matching columns.

  5. When editing is complete, save the file as CSV UTF-8.

  6. After editing, go back to the app and use Settings → Import Master CSV.

  7. Verify that item names and theoretical stock display correctly after import.

3-2. File name and requirements

ItemDescription

3-3. Column Specifications (Column Names)

The first row is the header row. Copy and paste the single line below exactly as shown. All five column names are required.

joint_code,barcode,name,theoretical_stock,actual_stock_total
ℹ️
You can keep the same five columns even when Partner Code is OFF The default setting is Partner Code OFF. If you manage only your own codes, simply leave the joint_code column blank. If you also use partner part numbers, fill the joint_code column. If imported master data contains values in joint_code, the app automatically turns Partner Code ON at startup.
#ColumnTypeRequiredDescription
1 joint_code String Optional Partner part number. Leave this blank if you manage own codes only. Fill it only when partner-code mode is needed.
2 barcode String Required Own part number / barcode. This column is always required, even when Partner Code is OFF. Must be unique.
3 name String Required Item name. Enclose in double quotes if it contains commas.
4 theoretical_stock Integer Optional Theoretical (book) stock quantity. Enter 0 if unknown.
5 actual_stock_total Integer Optional Running actual count total. Recommend 0 for first inventory.

3-4. Example

# Row 1: Header (Required)
joint_code,barcode,name,theoretical_stock,actual_stock_total
# Row 2+: Data
ABC-001,4912345678904,Hex Bolt M6x20,500,0
ABC-002,4912345678911,Hex Nut M6,1000,0
,4912345678928,Spring Washer M6,800,0
XY-999,4912345678935,"Bolt Set, M4",200,0
⚠️
Save as UTF-8 Always save as UTF-8 encoding. Shift-JIS or other encodings may cause garbled characters. Use "Save As" -> "CSV UTF-8" in Excel.

3-5. Importing the master CSV

You do not need to place the CSV in a fixed folder. Import it directly from the Settings screen.

🔌 Method 1: Select a local file

  1. Save the CSV anywhere on the device, such as Downloads.

  2. In the app, tap Settings → Import Master CSV.

  3. Choose the CSV in the system file picker.

  4. The imported data is applied immediately.

☁️ Method 2: Google Drive

  1. Upload the CSV to Google Drive from your PC.

  2. In the app, tap Settings → Import Master CSV.

  3. Select the CSV directly from Google Drive in the file picker.

📧 Method 3: Email attachment

  1. Email the CSV file to yourself.

  2. Download the attachment on your phone.

  3. In the app, tap Settings → Import Master CSV.

  4. Select the downloaded file.

ℹ️
Master Update Note If you edit the CSV again, import the updated file once more. The internal master file will be replaced with the new contents.

Section 05

Basic Operation Flow

5-1. Standard inventory flow

Step 1: Set the location (shelf)

  1. Tap the Location Scan button (it turns yellow).

  2. Point the camera at the shelf QR code — it is read automatically and filled into the Location field.

  3. Alternatively, tap the Location field and type the shelf number manually.

ℹ️
The location persists across scans. Update it each time you move to a different shelf.

Step 2: Scan the product barcode

  1. Confirm the camera switch is ON.

  2. Point the camera at the product barcode — it scans automatically.

  3. If found in master, the item name and theoretical stock are filled in automatically.

  4. If partner-code mode is enabled and the same input could match both an own code and a partner code, a selection dialog is shown instead of auto-opening the wrong item.

  5. If the code is not in master, the "Not in Master" dialog appears (→ see 6-3).

Step 3: Enter the quantity

  1. The Qty field receives focus automatically.

  2. Enter the count using the keypad.

  3. Use the calculator feature for multi-location totals (→ see 5-2).

Step 4: Save

  1. Optionally enter a memo.

  2. Tap ENT or Save.

  3. A sound plays and "Saved." appears — you're done. Proceed to the next item.

5-2. Using the calculator

Useful when items are spread across multiple locations and you need to total them.

  1. Tap the Calc field (keypad switches to calc mode).

  2. Example: 30 on top shelf, 25 on bottom → Enter 30, tap Add → Enter 25, tap Add.

  3. Expressions work too: 3*10 → Add adds 30 to Qty.

  4. When the Qty total is correct, tap Save.

5-3. Clear

The Clear button resets the barcode, item name, quantity, memo, and calc log — ready for the next item. The Location field is NOT cleared.

ℹ️
To reset the location: Tap the Location Scan button, use CLR to erase it manually, or scan a new shelf code to overwrite it.
Section 06

Advanced Operations

6-1. Name search mode

When a barcode is unavailable or unreadable, search for items by keyword.

  1. Enable the Name Search toggle at the bottom of the screen.

  2. Type a keyword in the Item Name field (partial match, case-insensitive).

  3. Tap 🔍 or Search.

  4. Select the item from the results dialog.

  5. Enter quantity and save as normal.

ℹ️
After saving: With the Continuous toggle OFF, name-search mode turns off automatically after each save. Enable Continuous (→ 6-2) to keep it active.

6-2. Continuous search mode

With the Cont. toggle ON, name-search mode remains active after each save. Convenient for entering multiple items from a single shelf by name.

6-3. Registering new items

When an unregistered code is scanned, the "Not in Master" dialog appears.

  1. Tap "Yes (Register)".

  2. An auto-generated ID starting with 99999 is set in the barcode field. For manual input, you choose whether to register as Own Code or Partner Code.

  3. Focus moves to the Item Name field — type the name.

  4. Enter quantity and save. The item is now in master for future scans.

6-4. Editing an item name

  1. Scan (or search by name) to load the item.

  2. Tap Edit Name (button turns orange).

  3. Edit the name in the Item Name field.

  4. Save — master is updated with the new name.

6-5. Editing the registered total

  1. Scan the item to load it.

  2. Tap the Edit button next to "Registered".

  3. Enter the new value in the dialog and tap Apply.

⚠️
Editing the registered total only updates the master file. Historical records in scan_history.csv are not modified.

6-6. Choosing own code or partner code

When partner-code mode is ON, some inputs may have more than one interpretation.

  1. If the "Search Type" dialog appears, choose Own Code or Partner Code.

  2. The same confirmation flow is used during camera scanning when the code is ambiguous.

  3. If this happens frequently, review the code design in your master CSV.

7-1. Viewing save history

The Save History bar at the bottom of the screen shows recent records (newest first): time, item name, quantity, and theoretical stock.

Tap the History button for the full history dialog, which also shows location and memo.

Every save appends a row to scan_history.csv. Transfer this file to a PC to analyze in Excel or Google Sheets.

Section 08

Settings (Language, Rules, Camera)

Tap the ⚙️ icon in the header to open the settings dialog.

8-1. Import Master CSV

Select a CSV from your device, Downloads folder, or cloud storage to update the internal master data. The imported data is applied immediately, so no separate reload action is needed.

8-2. Clear All History

Deletes scan_history.csv and resets the history completely. Run this at the end of each inventory cycle.

🚨
This action cannot be undone. Always back up the CSV file before proceeding.

8-3. Use Partner Code

When ON, a dedicated Partner Code field appears alongside the Own Code field. Use this when you need to track your trading partner's part numbers alongside your own.

StateBehavior
OFF (default)Partner code field hidden. Manage with own codes only.
ONPartner code field displayed. When entering a code, a dialog asks whether it is an Own Code or Partner Code.
ℹ️
If the master CSV contains data in the joint_code column, this setting turns ON automatically at startup.

From the Settings screen, you can import a master CSV, share the current history and master files, and export all data.

Section 10

Troubleshooting

Barcode won't scan / no response
  • Make sure the camera switch (top right) is ON.
  • Keep the code centered on screen (recognition drops near the edges).
  • Check that the correct scan format (CODE39 / EAN13 etc.) is enabled in Settings.
  • Damaged or dirty barcodes may not scan — use manual entry.
  • Verify camera permission: Settings → Apps → Quantiventra → Permissions → Camera.
Master CSV placed on device but item names aren't showing
  • Filename must be exactly quantiventra_master.csv (all lowercase).
  • Make sure you selected the correct file through Settings → Import Master CSV.
  • If you want to use a different CSV, run Import Master CSV again from Settings and select the file.
  • Confirm the file is saved as UTF-8 (not Shift-JIS or Windows-1252).
  • Check that the header row matches: joint_code,barcode,name,theoretical_stock,actual_stock_total
Can't save / "Save failed" message
  • Ensure the device has sufficient free storage.
  • A barcode or item name must be entered before saving.
Location Scan pressed but shelf number isn't set
  • Check that the location scan format is correct (default: QRCODE).
  • If your shelf labels use a different format, change the Location code type in Settings.
  • Make sure the camera switch is ON.
I cannot find the exported files

Check the folder you selected during export. The files are named scan_history_[timestamp].csv and quantiventra_master_export.csv.

Edit Name button pressed but the name field isn't editable
  • An item must be loaded first (via scan or name search). The button has no effect when no item is loaded.
  • The button turns orange when active — if it's still grey, tap it again.
ENT in the Calc field gives 0 / calculation doesn't apply
  • Use the Add key to add the calc result to Qty, then save. Pressing ENT from the Calc field does not save.
  • Full-width (double-byte) characters in expressions cause errors — use the on-screen keypad.
  • Expressions cannot start with an operator (e.g. *5 is invalid; use 3*5).
