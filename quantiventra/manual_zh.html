Section 01

简介

1-1. 关于本应用

Quantiventra（"Quantity（数量）"与"Inventory（库存/盘点）"的合成词）是一款利用智能手机摄像头进行条码扫描，从而提升盘点和库存清点效率的Android应用。

将摄像头对准条码即可自动识别商品，输入数量后保存，轻松实现盘点作业数字化。扫描结果以CSV文件形式保存到设备，可与Excel或Google表格无缝衔接。

📦 库存盘点

通过条码扫描快速定位商品，即时记录数量。内置计算功能，多处合计轻松搞定。

🗂️ 主数据管理

通过CSV文件批量导入主数据，预先配置品名、品号和理论库存。

📍 货位管理

扫描QR码设置货位，CSV中自动记录每次盘点的货位信息。

📊 CSV导出

扫描结果自动保存到设备存储，传至PC后可进行汇总与分析。

1-2. 运行环境

项目要求
系统Android 7.0（Nougat / API 24）及以上
摄像头支持自动对焦的摄像头（必须）
存储建议剩余50MB以上
权限摄像头权限
网络不需要（完全离线运行）

1-3. 关于权限

应用首次启动时会请求以下权限。若要正常使用扫码功能，只需授予摄像头权限。

权限用途拒绝后的影响
摄像头扫描条码和QR码无法使用摄像头扫码（仍可手动输入）
ℹ️

Section 03

主数据CSV的创建与导入【重要】

✅
提前准备主数据可大幅提升作业效率 导入主数据后，扫描条码时会自动显示品名和理论库存。如果未登记，则每次都需要手动输入。

3-1. 准备主数据CSV的两种方法（含列名）

您可以通过以下两种主要方法来准备主数据文件。

方法A：使用Excel或Google表格从零创建

  1. 在电脑上打开Excel或Google表格。

  2. 不要手动输入第1行表头。请直接复制本手册 3-3 字段说明（列名） 中的整行内容并粘贴。

  3. 从第二行开始输入您的商品数据。

  4. 另存为“CSV（逗号分隔）”格式，文件名必须为 quantiventra_master.csv（推荐使用UTF-8编码）。

方法B：使用应用自动生成的文件进行编辑（推荐初学者）

如果您觉得从头创建比较麻烦，可以先让应用为您生成一个模板。

  1. 在没有主数据文件的情况下启动应用，扫描任意商品的条码。

  2. 弹出“未登记品号”对话框时，点击“是（新建登记）”。

  3. 在品名处输入“测试商品”等临时名称并保存。

  4. 接着在应用中执行 设置 → 导出全部数据，即可导出当前主数据CSV。

  5. 将导出的文件带到电脑（通过USB、邮件或云端均可），用Excel打开，并参考测试商品的数据格式填写正式数据。完成后，再通过 导入主数据CSV 导回应用。

3-2. 文件名和要求

项目要求
文件名quantiventra_master.csv（全部小写，固定不变）
字符编码UTF-8（推荐无BOM格式）
换行符LF 或 CRLF
导入方式从 设置 → 导入主数据CSV 选择文件

3-3. 字段说明（列名）

第1行是表头行。请将下面这一整行原样复制并粘贴。5个列名都必须保留。

joint_code,barcode,name,theoretical_stock,actual_stock_total
ℹ️
即使只使用自社品号，也请保持这5列不变 应用默认是对方品号 OFF。如果您只管理自社品号，可以将 joint_code 列留空；只有在同时管理对方品号时才填写该列。如果导入的主数据中 joint_code 列含有数据，应用启动时会自动切换为 ON。
#列名类型必填说明
1 joint_code 字符串 可选 对方品号（不使用时留空）。仅限英数字、连字符、下划线。
2 barcode 字符串 必填 自有品号 / 条码。必须唯一，不可重复。
3 name 字符串 必填 品名。若包含逗号需用双引号括起来。
4 theoretical_stock 整数 可选 理论库存数量。若未知请输入 0。
5 actual_stock_total 整数 可选 累计实际盘点数量。首次盘点建议填 0。

3-4. 示例

# 第1行：表头（必填）
joint_code,barcode,name,theoretical_stock,actual_stock_total
# 第2行起：数据行
ABC-001,4912345678904,六角螺栓M6x20,500,0
ABC-002,4912345678911,六角螺母M6,1000,0
,4912345678928,弹簧垫圈M6,800,0
XY-999,4912345678935,"螺栓套装,M4",200,0
⚠️
保存时的注意事项 使用Excel保存时，请选择“另存为” -> “CSV UTF-8 (逗号分隔)”。使用其他编码可能导致乱码。

3-5. 导入主数据CSV

无需把CSV放到固定文件夹中。请直接从设置画面导入。

🔌 方法①：选择本地文件

  1. 将CSV保存在设备上的任意位置，例如“下载”。

  2. 在应用中点击设置 → 导入主数据CSV。

  3. 在系统文件选择器中选中CSV文件。

  4. 导入完成后，主数据会立即生效。

☁️ 方法②：Google云端硬盘

  1. 在电脑上将CSV上传到 Google Drive。

  2. 在应用中点击设置 → 导入主数据CSV。

  3. 在文件选择器中直接从Google Drive选择该CSV。

📧 方法③：邮件附件

  1. 先把CSV通过邮件发送给自己。

  2. 在手机上下载附件。

  3. 在应用中点击设置 → 导入主数据CSV，然后选择下载的文件。

ℹ️
更新主数据时 如果你再次修改了CSV，请重新导入更新后的文件。应用内部保存的主数据会被新文件内容替换。

Section 05

基本操作流程

5-1. 标准盘点流程

第1步：设置货位

  1. 点击货位扫描按钮（按钮变为黄色）。

  2. 将摄像头对准货架QR码，自动读取并填入货位（LOC）字段。

  3. 也可点击货位字段直接手动输入货架编号。

ℹ️
货位设定后，在扫描下一件商品时保持不变。移动到其他货架时请及时更新。

第2步：扫描商品条码

  1. 确认摄像头开关已开启。

  2. 将摄像头对准商品条码，自动扫描。

  3. 若主数据中已登记，品名和理论库存将自动显示。（优先搜索对方品号，再搜索自有品号。）

  4. 若为未登记条码，将显示"未登记品号"对话框（→参照6-3）。

第3步：输入数量

  1. 数量字段自动获得焦点。

  2. 通过数字键盘输入数量。

  3. 如需多处合计，请使用计算功能（→参照5-2）。

第4步：保存

  1. 根据需要输入备注。

  2. 点击确认或保存。

  3. 播放提示音并显示"已保存"即完成。继续扫描下一件商品。

5-2. 计算功能的使用方法

适用于商品分散在多处货架、需要合计数量的场景。

  1. 点击计算字段（键盘切换为计算模式）。

  2. 示例：上层30个、下层25个 → 输入30点累加 → 输入25点累加。

  3. 也支持表达式：3*10 → 点累加，数量增加30。

  4. 数量合计无误后，点击保存。

5-3. 清除

点击清除按钮，将重置条码、品名、数量、备注和计算记录，准备好录入下一件商品。货位（LOC）不会被清除。

ℹ️
如需重置货位：点击货位扫描按钮，用CLR手动清除，或扫描新货架的条码覆盖原值。
Section 06

高级操作

6-1. 品名搜索模式

当条码不可用或无法扫描时，可通过关键字按品名搜索商品。

  1. 开启屏幕底部的品名搜索开关。

  2. 在品名字段输入关键字（支持部分匹配，全半角不区分）。

  3. 点击🔍或搜索。

  4. 从搜索结果对话框中点击选择对应商品。

  5. 之后按常规方式输入数量并保存。

ℹ️
保存后的动作：关闭"连续"开关时，每次保存后品名搜索模式自动关闭。如需连续使用，请开启"连续"开关（→参照6-2）。

6-2. 连续搜索模式

开启连续开关后，保存后品名搜索模式将保持开启状态。适合在同一货架连续按品名录入多件商品。

6-3. 新商品登记

扫描到主数据中未登记的条码时，将显示"未登记品号"对话框。

  1. 点击"是（新建登记）"。

  2. 系统自动分配以99999开头的编号填入条码字段。手动输入时，可选择登记为自有品号或对方品号。

  3. 焦点自动移至品名字段，输入品名。

  4. 输入数量后保存。下次扫描时将作为已登记商品识别。

6-4. 修改品名

  1. 扫描（或通过品名搜索）加载目标商品。

  2. 点击修改品名按钮（按钮变为橙色）。

  3. 品名字段变为可编辑状态，进行修改。

  4. 保存后主数据以新品名更新。

6-5. 修改已登记累计

  1. 扫描目标商品并加载。

  2. 点击"已登记"旁边的修改按钮。

  3. 在对话框中输入新值，点击"修改"。

⚠️
修改已登记累计只更新主数据文件中的累计值，不会修改scan_history.csv中的历史记录。

Section 10

故障排除

条码无法扫描 / 没有反应
  • 确认右上角摄像头开关已开启。
  • 请将条码保持在画面中央（靠近边缘时识别率会下降）。
  • 确认设置中已启用正确的扫描格式（CODE39 / EAN13等）。
  • 条码破损或污脏时可能无法扫描，请使用手动输入。
  • 确认摄像头权限已授予：设置 → 应用 → Quantiventra → 权限 → 摄像头。
已放入主数据CSV但品名未显示
  • 确认所选择的CSV内容正确，并且标题行格式无误。
  • 如果要切换到别的CSV，请从设置画面再次执行 导入主数据CSV 并选择文件。
  • 确认CSV编码为UTF-8（使用其他编码会乱码）。
  • 确认标题行格式正确：joint_code,barcode,name,theoretical_stock,actual_stock_total。
无法保存 / 显示"保存失败"
  • 摄像头权限仅用于扫码；若只进行手动输入，则无需开启摄像头。
  • 确认设备存储空间充足。
  • 条码和品名均未输入时无法保存。
点击货位扫描按钮后货架编号未设置
  • 确认货位用扫描格式设置正确（默认为QRCODE）。
  • 若货架标签使用其他格式，请在设置中修改货位扫描类型。
  • 确认摄像头开关已开启。
每次启动应用时都会打开存储权限设置页

当前版本通常不需要这种权限页面。若系统仍然弹出相关设置，请先返回应用；一般使用只需要摄像头权限。
