Section 01

소개

1-1. 본 앱에 대하여

Quantiventra(콴티벤트라)는 'Quantity(수량)'와 'Inventory(재고·실사)'를 조합한 이름으로, 스마트폰 카메라를 이용한 코드 스캔을 통해 재고 조사 및 카운트 작업을 효율화하는 Android 앱입니다.

카메라를 코드에 비추기만 하면 상품을 자동으로 인식하고, 수량을 입력하여 저장하는 직관적인 조작으로 종이 기반의 재고 조사 작업을 디지털화할 수 있습니다. 스캔 결과는 CSV 파일로 기기에 저장되므로 Excel이나 Google 스프레드시트와의 연동도 간편합니다.

📦 재고 카운트

코드 스캔으로 상품을 빠르게 특정하고 수량을 즉시 기록. 계산 기능 내장으로 여러 장소의 합산도 간단.

🗂️ 마스터 관리

CSV 파일로 마스터 데이터를 일괄 등록. 품명·품번·이론 재고를 미리 설정할 수 있습니다.

📍 로케이션 관리

QR 코드로 선반 번호(로케이션)를 스캔 설정. 어느 선반의 재고인지 CSV에 기록합니다.

📊 CSV 출력

스캔 결과는 기기 저장소에 자동 저장. PC로 가져와서 집계 및 분석 가능.

Section 02

설치 및 초기 설정

2-1. 설치 절차

  1. Android 기기에서 Google Play 스토어를 엽니다.

  2. 검색창에 "Quantiventra"를 입력하여 검색합니다.

  3. 앱을 선택하고 설치를 탭합니다.

  4. 설치 완료 후 열기를 탭하거나 홈 화면의 아이콘을 눌러 앱을 실행합니다.

2-2. 최초 실행 및 권한 허용

  1. 앱 실행 후 카메라 권한 대화상자가 표시되면 허용을 탭해 주십시오.

  2. 권한을 허용하면 바로 스캔을 시작할 수 있습니다.

  3. 마스터 CSV를 사용하려면 설정 → 마스터 CSV 가져오기에서 파일을 선택해 주십시오.

ℹ️
초기 설정 후 카메라 권한만 허용하면 스캔, 저장, 마스터 CSV 가져오기, 데이터 내보내기를 모두 설정 화면에서 진행할 수 있습니다.

2-3. 데이터가 관리되는 방식

마스터, 이력, 설정 데이터는 앱이 자동으로 관리합니다. 일반적인 사용에서는 내부 폴더 구조를 알거나 직접 다룰 필요가 없습니다.

ℹ️
사용자가 해야 할 일 마스터 데이터를 불러올 때는 마스터 CSV 가져오기를, 데이터를 앱 밖으로 꺼낼 때는 전체 데이터 내보내기 또는 이력 공유를 사용해 주십시오.
Section 03

마스터 CSV 작성 및 등록 【중요】

✅
마스터를 미리 준비하면 작업 효율이 대폭 향상됩니다 마스터 CSV를 등록해 두면 코드 스캔 시 품명·이론 재고가 자동 표시됩니다. 미등록 상태인 경우 매번 수동으로 입력해야 합니다.

3-1. 마스터 CSV 준비 방법 (열 이름 포함, 2가지 방법)

마스터 파일을 준비하는 방법은 크게 두 가지가 있습니다.

방법 A: Excel이나 스프레드시트에서 처음부터 작성하기

  1. PC에서 Excel 또는 Google 스프레드시트를 엽니다.

  2. 첫 번째 줄 헤더는 직접 입력하지 말고, 이 매뉴얼의 3-3 컬럼 사양 (열 이름)에 있는 한 줄을 그대로 복사해서 붙여넣으십시오.

  3. 두 번째 줄부터 상품 데이터를 입력합니다.

  4. 'CSV(쉼표로 분리)' 형식으로 파일 이름을 quantiventra_master.csv로 저장합니다(UTF-8 인코딩 권장).

방법 B: 앱에서 자동 생성된 파일을 기반으로 하기 (초보자 권장)

처음부터 만들기 번거로운 경우 앱이 견본을 만들게 할 수 있습니다.

  1. 마스터 파일이 없는 상태에서 앱을 실행하여 아무 상품의 바코드를 스캔합니다.

  2. '미등록 코드' 대화상자가 나타나면 '예(신규 등록)'를 선택합니다.

  3. 품명에 '테스트' 등을 입력하고 저장합니다.

  4. 이후 앱에서 설정 → 전체 데이터 내보내기를 실행하면 현재 마스터 CSV를 내보낼 수 있습니다.

  5. 내보낸 파일을 PC로 가져와(USB, 메일, 클라우드 등) Excel에서 엽니다. 테스트 데이터 형식을 참고해 실제 데이터를 추가·수정한 뒤, 앱의 마스터 CSV 가져오기로 다시 불러오십시오.

3-2. 파일 이름과 저장 위치

항목값
파일 이름quantiventra_master.csv (고정·변경 불가)
문자 코드UTF-8 (BOM 없음 권장)
줄 바꿈 코드LF 또는 CRLF (둘 다 가능)
가져오는 방법설정 → 마스터 CSV 가져오기에서 파일 선택

3-3. 컬럼 사양 (열 이름)

첫 번째 줄은 헤더 행입니다. 아래 한 줄을 그대로 복사해서 붙여넣으십시오. 5개의 열 이름이 모두 필요합니다.

joint_code,barcode,name,theoretical_stock,actual_stock_total
ℹ️
자사 품번만 사용하는 경우에도 같은 5개 열을 그대로 사용합니다 앱의 기본 설정은 상대방 품번 OFF입니다. 자사 품번만 관리할 경우 joint_code 열은 비워 두면 됩니다. 상대방 품번도 함께 관리할 때만 값을 입력하십시오. 가져온 마스터 CSV의 joint_code 열에 데이터가 있으면, 앱 시작 시 자동으로 ON으로 전환됩니다.
#컬럼명타입필수설명·주의사항
1 joint_code 문자열 선택 거래처 품번(상대방 코드). 빈칸 가능. 영문자·숫자·하이픈·언더바만 가능(설정에 따름).
2 barcode 문자열 필수 자사 코드 번호. 실제 코드와 일치해야 함. 중복 불가.
3 name 문자열 필수 품명. 쉼표(,)를 포함하는 경우 따옴표로 감쌈 (예: "볼트,M4").
4 theoretical_stock 정수 선택 이론 재고수(장부 재고). 모를 경우 0 입력.
5 actual_stock_total 정수 선택 등록된 누적 재고수. 재고 조사 시작 시에는 0을 권장.

3-4. CSV 샘플

# 1행: 헤더 행 (필수)
joint_code,barcode,name,theoretical_stock,actual_stock_total
# 2행 이후: 데이터 행
ABC-001,4912345678904,육각볼트M6×20,500,0
ABC-002,4912345678911,육각너트M6,1000,0
,4912345678928,스프링와셔M6,800,0
XY-999,4912345678935,"볼트세트,M4",200,0
⚠️
Excel에서 편집 시 주의사항 Excel에서 저장하면 문자 코드가 Shift-JIS가 될 수 있습니다. '다른 이름으로 저장' → 'CSV UTF-8(쉼표로 분리)'을 선택하십시오. 파일 맨 앞의 BOM이 문제가 될 경우 메모장 등으로 다시 열어 저장하십시오.

3-5. 마스터 CSV 가져오기

CSV를 고정 폴더에 넣을 필요는 없습니다. 설정 화면에서 직접 가져오면 됩니다.

🔌 방법 ①: 로컬 파일 선택

  1. CSV 파일을 기기의 임의 위치(예: 다운로드)에 저장합니다.

  2. 앱에서 설정 → 마스터 CSV 가져오기를 탭합니다.

  3. 시스템 파일 선택기에서 CSV 파일을 선택합니다.

  4. 가져오기가 끝나면 마스터 데이터가 즉시 반영됩니다.

☁️ 방법 ②: Google Drive에서 선택

  1. PC에서 CSV를 Google Drive에 업로드합니다.

  2. 앱에서 설정 → 마스터 CSV 가져오기를 탭합니다.

  3. 파일 선택기에서 Google Drive의 CSV를 직접 선택합니다.

📧 방법 ③: 메일 첨부파일 사용

  1. CSV 파일을 본인에게 메일로 보냅니다.

  2. 스마트폰에서 첨부파일을 다운로드합니다.

  3. 앱으로 돌아와 설정 → 마스터 CSV 가져오기를 탭한 뒤 다운로드한 파일을 선택합니다.

ℹ️
마스터를 수정한 뒤에는 CSV를 다시 편집했다면 업데이트된 파일을 한 번 더 가져와 주십시오. 앱 내부의 마스터 데이터는 새 파일 내용으로 교체됩니다.
Section 04

화면 구성 및 각부 설명

메인 화면은 위에서부터 '헤더 바', '카메라 영역', '입력 폼 영역', '저장 이력 영역', '숫자 키패드'의 5개 영역으로 구성되어 있습니다.

4-1. 헤더 바

요소설명
앱 이름 (Quantiventra)탭해도 동작 없음 (표시 전용)
⚙️ 설정 아이콘설정 대화상자를 엶
ON / OFF 스위치카메라 프리뷰 켜기/끄기 전환

4-2. 카메라 영역

코드·QR 코드를 실시간으로 스캔하는 영역입니다. 카메라를 켠 상태에서 코드를 카메라에 비추면, 3프레임 연속으로 같은 코드를 읽었을 때 자동으로 인식됩니다.

ℹ️
스캔 정밀도에 대하여 카메라는 화면 중앙 ±15% 범위에 들어온 코드만 우선 인식합니다. 인식이 잘 안 될 경우 코드를 화면 중앙에 맞추어 주십시오.

수량이번에 카운트한 수량을 입력. 숫자 키패드로 입력하며 계산식(예: 3*5)도 사용 가능.

비고 (메모)임의의 메모를 기록. CSV에 저장됨.

Section 05

기본 조작 흐름

5-1. 표준적인 재고 조사 흐름

단계 1: 로케이션(선반 번호) 설정하기

  1. 로케이션을 설정할 경우: 로케이션 읽기 버튼을 탭합니다 (버튼이 노란색으로 변함).

  2. 로케이션 코드에 카메라를 비추면 자동 인식되어 '로케이션 (LOC)' 필드에 반영됩니다.

  3. 수동 입력 시: '로케이션 (LOC)' 필드를 탭하여 포커스한 후 소프트 키보드로 입력합니다.

ℹ️
로케이션은 한 번 설정하면 다음 상품 스캔 시에도 유지됩니다. 선반을 이동했을 때 매번 업데이트해 주십시오.

단계 2: 상품 코드를 스캔하기

  1. 카메라 스위치가 ON인지 확인합니다.

  2. 상품 코드에 카메라를 비춥니다. 자동으로 스캔됩니다.

  3. 마스터 등록 상품인 경우 품명·이론 재고가 자동 표시됩니다. (상대방 품번 → 자사 품번 순으로 검색합니다.)

  4. 미등록 코드인 경우 '미등록 코드' 대화상자가 표시됩니다 (→ 6-3 참조).

단계 3: 수량 입력하기

  1. '수량' 필드에 자동으로 포커스가 이동합니다.

  2. 숫자 키패드로 수량을 입력합니다.

  3. 여러 위치의 합산이 필요한 경우 계산 기능을 이용합니다 (→ 5-2 참조).

단계 4: 저장하기

  1. 필요에 따라 메모를 입력합니다.

  2. ENT 키 또는 저장 키를 탭합니다.

  3. 효과음이 울리고 '저장했습니다' 메시지가 표시되면 완료입니다.

  4. 다음 상품 스캔으로 진행합니다.

5-2. 계산 기능 사용법

여러 층이나 장소에 나뉘어 있는 상품을 카운트할 때 편리한 기능입니다.

  1. '계산용' 필드를 탭하여 포커스합니다 (숫자 키패드가 계산 모드로 전환됨).

  2. 예: 선반 상단에 30개, 하단에 25개 있는 경우 → 30 입력 후 가산 → 25 입력 후 가산

  3. 계산식 사용 가능: 3*10 → 가산을 누르면 30이 수량에 더해집니다.

  4. 계산 이력 칸에 가산된 내역이 표시됩니다.

  5. 합계가 수량 칸에 반영되면 저장을 탭합니다.

5-3. 클리어 (초기화)

클리어 버튼을 탭하면 현재 입력 내용(코드·품명·수량·메모·계산 이력)을 모두 리셋하고 다음 상품 입력이 가능한 상태로 돌아갑니다. 로케이션(LOC)은 초기화되지 않습니다.

ℹ️
로케이션(LOC)을 리셋하고 싶은 경우 로케이션은 클리어 버튼으로 지워지지 않습니다. 다른 선반으로 이동할 때는 로케이션 읽기 버튼을 눌러 CLR 키로 수동 삭제하거나, 새로운 로케이션 코드를 스캔하여 덮어쓰십시오.
Section 06

응용 조작

6-1. 품명 검색 모드

코드가 없거나 읽을 수 없을 때 품명 키워드로 상품을 검색할 수 있습니다.

  1. 화면 하단의 '품명 검색' 스위치를 ON으로 설정합니다.

  2. '품명' 필드에 키워드를 입력합니다 (부분 일치·전각 반각 구분 없음).

  3. 🔍 검색 버튼 또는 검색(확정)을 탭합니다.

  4. 검색 결과 대화상자가 표시되면 해당 상품을 탭하여 선택합니다.

  5. 이후에는 평소와 같이 수량을 입력하여 저장합니다.

ℹ️
저장 후 품명 검색 모드의 동작 '연속' 스위치가 OFF인 경우, 상품을 선택하여 저장하면 품명 검색 모드가 자동으로 꺼집니다. 다음번에는 코드 스캔 대기 상태인 일반 상태로 돌아갑니다. 계속해서 품명 검색을 사용하려면 '연속' 스위치를 ON으로 설정하십시오 (→ 6-2 참조).

6-2. 연속 검색 모드

'연속' 스위치를 ON으로 설정하면 저장 후에도 품명 검색 모드가 유지됩니다. 동일한 선반의 여러 상품을 품명으로 연속 입력할 때 편리합니다.

6-3. 신규 상품 등록

마스터에 등록되지 않은 코드를 스캔한 경우 '미등록 코드' 대화상자가 표시됩니다.

  1. 대화상자에서 '예(신규 등록)'를 탭합니다.

  2. 자동 채번된 자사 ID가 코드 칸에 설정됩니다 (99999로 시작하는 번호). 수동 입력으로 신규 등록하는 경우에는 상대방 품번인지 자사 품번인지 선택하게 됩니다.

  3. 품명 필드로 포커스가 이동하면 품명을 입력합니다.

  4. 수량을 입력하여 저장합니다. 다음번부터는 마스터 등록 상품으로 인식됩니다.

6-4. 품명 변경

마스터에 이미 등록된 상품의 품명을 수정할 수 있습니다.

  1. 대상 상품을 스캔(또는 품명 검색으로 선택)합니다.

  2. 품명 변경 버튼을 탭합니다 (버튼이 주황색으로 변함).

  3. 품명 필드가 편집 가능하게 바뀌면 수정합니다.

  4. 저장하면 새로운 품명으로 마스터가 업데이트됩니다.

6-5. 등록됨 누적 수정

과거의 재고 데이터를 수정해야 하거나 초기값을 설정할 때 사용합니다.

  1. 대상 상품을 스캔하여 불러옵니다.

  2. '등록됨' 옆에 있는 변경 버튼을 탭합니다.

  3. 대화상자에서 새로운 값을 입력하고 '변경'을 탭합니다.

⚠️
등록됨 누적을 변경해도 scan_history.csv의 과거 이력은 변경되지 않습니다. 어디까지나 마스터의 누적값만 변경됩니다.

Section 10

문제 해결(트러블슈팅)

코드가 읽히지 않거나 반응이 없음
  • 카메라 스위치(오른쪽 상단)가 ON으로 되어 있는지 확인하십시오.
  • 코드를 화면 중앙에 맞추십시오 (가장자리로 치우치면 인식률이 떨어집니다).
  • 설정에서 스캔 대상 종류(CODE39 / EAN13 등)가 해당 코드와 일치하는지 확인하십시오.
  • 코드 인쇄 상태가 불량하거나 오염된 경우 읽지 못할 수 있습니다.
  • 카메라 권한이 허용되어 있는지 확인하십시오: 설정 → 앱 → Quantiventra → 권한 → 카메라.
마스터 CSV를 배치했지만 품명이 표시되지 않음
  • 선택한 CSV 파일의 내용이 올바른지, 그리고 헤더 행이 맞는지 확인하십시오.
  • 다른 CSV로 바꾸고 싶다면 설정 화면에서 마스터 CSV 가져오기를 다시 실행해 파일을 선택하십시오.
  • CSV의 문자 코드가 UTF-8인지 확인하십시오 (Shift-JIS인 경우 글자 깨짐이 발생합니다).
  • 헤더 행 형식이 올바른지 확인하십시오 (joint_code,barcode,name,theoretical_stock,actual_stock_total).
저장되지 않음 / '저장 실패' 메시지가 뜸
  • 카메라 권한은 스캔용이며, 수동 입력만 사용할 경우 카메라를 켜지 않아도 됩니다.
  • 기기의 여유 공간이 부족하지 않은지 확인하십시오.
  • 코드와 품명이 모두 입력되지 않은 상태에서는 저장할 수 없습니다.
로케이션 읽기 버튼을 눌렀지만 로케이션이 설정되지 않음
  • 로케이션용 코드 종류 설정이 올바른지 확인하십시오 (기본값은 QRCODE).
  • 로케이션용 코드가 QR 코드가 아닌 바코드인 경우, 설정에서 로케이션용 종류를 변경하십시오.
  • 카메라 스위치가 ON인지 확인하십시오.
앱 실행 시 저장소 설정 화면이 열림

현재 버전에서는 보통 이런 권한 화면이 필요하지 않습니다. 만약 시스템이 관련 화면을 열더라도 일반적인 사용에는 카메라 권한만 있으면 됩니다.

품명 변경 버튼을 눌러도 품명 필드를 편집할 수 없음
  • 코드 또는 품명 검색으로 상품을 불러온 상태인지 확인하십시오. 아무것도 불러오지 않은 상태에서는 품명 변경이 기능하지 않습니다.
  • 품명 변경 버튼을 탭하면 버튼이 주황색으로 변합니다. 주황색이 되지 않았다면 탭이 인식되지 않았을 가능성이 있으니 다시 한번 탭하십시오.
계산용 칸에서 ENT를 눌러도 결과가 0이 됨 / 계산이 반영 안 됨
  • '가산' 키를 눌러 수량 칸에 더한 후에 저장하십시오. 계산용 칸 상태에서 ENT를 눌러도 저장이 수행되지 않습니다.
  • 계산식에 전각 문자(전각 숫자·기호)가 포함되면 오류가 발생합니다. 숫자 키패드를 사용하여 입력하십시오.
  • 계산식 맨 앞이 연산자(예: *5)인 경우 오류가 발생합니다. 숫자부터 입력하십시오 (예: 3*5).
